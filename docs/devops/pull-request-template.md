## Introdução

Este é o template padrão que deve ser seguido em todas as submissões de código via Pull Request.

---

## Template

```
## 📌 Descrição
Explique de forma clara e objetiva:
- O que foi alterado?
- Por que foi alterado?
- Contexto adicional, se necessário.

---

## 📝 Tipo de Mudança
Selecione a opção que melhor descreve a PR:

- [ ] 🐛 Bugfix (correção de bug)
- [ ] ✨ Nova feature (funcionalidade)
- [ ] 🔧 Refactor (ajustes de código sem mudar funcionalidade)
- [ ] 🧹 Chore (tarefas de manutenção, CI/CD, configs)
- [ ] 📖 Documentação
- [ ] 🚀 Performance

---

## 🔗 Issue Relacionada
- Closes/Fixes #[Número da Issue]  
- Links adicionais, se houver.

---

## ✅ Checklist de Qualidade
Antes de pedir revisão, verifique:

- [ ] Código compila sem erros ⚡
- [ ] Testes adicionados ou atualizados 🧪
- [ ] Testado em ambientes (dev/staging/prod) 🌐
- [ ] Documentação atualizada (README, Wiki, etc.) 📚
- [ ] Segue os padrões de código e lint definidos 🛠️
- [ ] Nenhum warning ou erro no build 🔍
- [ ] Screenshots/Logs adicionados (se aplicável) 🖼️

---

## 📷 Evidências
Adicione prints de tela, logs ou GIFs que mostrem a mudança em ação.

---

## 🔄 Impacto Esperado
- Como isso afeta o usuário/cliente?
- Existe risco de quebra?
- Qual o plano de rollback, se necessário?

---

## ⚙️ Ambientes de Teste
Liste onde foi validado:

- [ ] Dev  
- [ ] Staging  
- [ ] Produção  

---

## 📦 Dependências
Liste dependências externas ou pacotes adicionados/alterados.

---

## 🗒️ Notas Extras
Comentários adicionais para os revisores ou para o time DevOps.
```
